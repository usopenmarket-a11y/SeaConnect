## Part 1: Boat Owner Onboarding

### 1.1 Owner Acquisition Strategy (Pre-Launch)

**Target:** 50+ verified, active boats before public launch.

**Channels:**
| Channel | Approach | Target Count |
|---------|----------|-------------|
| Direct outreach — Hurghada marinas | Operations team visits Red Sea marina offices, pitches SeaConnect | 20 boats |
| Direct outreach — North Coast | Visit marina offices in Sahel, Alexandria coast | 10 boats |
| WhatsApp groups (fishing & boating) | Founder or rep joins existing groups, shares platform | 10 boats |
| Facebook groups (قوارب وصيد مصر) | Organic posts, personal invitations | 5 boats |
| Referral from first 10 owners | Offer 1-month zero-commission incentive for referrals | 5+ boats |

**Incentive for first 50 owners:**
- First 3 months: reduced commission (8% instead of 12%)
- "Founding Partner" badge on profile permanently
- Free featured listing for 30 days at launch

### 1.2 Owner Onboarding Flow (App)

```
Step 1: Download app → Register → Select "I'm a Boat Owner"
    → Enter: name, phone, email, password
    → OTP verification
    → Role confirmed as OWNER

Step 2: Complete Owner Profile
    → Full name / business name
    → National ID number (required for KYC)
    → Bank account details (for payouts via Fawry)
    → Profile photo
    → Operating location(s)

Step 3: Create First Listing
    → Boat name, type (fishing/yacht/speedboat)
    → Description (Arabic/English)
    → Capacity (max passengers)
    → Price per day / half-day / hour
    → Trip types offered (full day, half day, sunset, overnight)
    → Departure port / marina

Step 4: Upload Photos & Video
    → Minimum 3 photos required (enforced)
    → Up to 10 photos
    → Up to 1 video (50MB max)
    → Photo tips shown in UI (good lighting, show full boat + interior + deck)

Step 5: Upload Documents (required for approval)
    → Vessel Registration Certificate (photo/PDF)
    → Marine Insurance Certificate
    → Captain's License (if owner operates)
    → Tourism Activity License (if applicable)

Step 6: Set Availability Calendar
    → Block out unavailable dates (maintenance, personal use, etc.)
    → Default: all dates available unless blocked

Step 7: Submit for Admin Review
    → "Your listing is under review. We'll notify you within 24 hours."
    → Admin reviews documents + photos in admin portal
    → Approve: listing goes live, owner notified via push + email
    → Reject: reason sent to owner, they can correct and resubmit
```

### 1.4 Owner Operations Guide

**Responding to Bookings:**
- You have **2 hours** to accept or decline a booking request
- If you don't respond, the booking auto-declines and the customer is refunded
- Set your notification settings to "always on" for booking alerts

**Managing Your Calendar:**
- Block dates you know you're unavailable (storms, maintenance, etc.)
- Keep your calendar current — a customer-confirmed booking cannot be cancelled without a penalty

**Receiving Payouts:**
- Payout is sent **24 hours after the trip is completed**
- You receive: trip price minus 12% SeaConnect commission minus ~2.5% Fawry fee
- View your earnings in the "My Earnings" tab in the app

**Handling No-Shows:**
- If the customer doesn't show up, mark the trip as a "no-show" in the app
- No-show policy: customer is not refunded if they cancel within 24 hours of departure
- Contact support if there's a dispute: support@seaconnect.app

### 2.2 Vendor Onboarding Flow (App)

```
Step 1: Register → Select "I'm a Vendor"
    → Name, phone, email, password → OTP

Step 2: Store Profile
    → Store name (will appear to customers)
    → Store description (Arabic/English)
    → Store logo (required — enforced)
    → Operating location / city
    → Business registration number (optional at MVP, required for payout)
    → Bank account / Fawry wallet for payouts

Step 3: Create Product Listings
    → Product name (Arabic + English)
    → Description
    → Category (Fishing Rods / Reels / Lures / Clothing / Accessories / Boats & Equipment)
    → Price (EGP)
    → Stock quantity
    → Up to 5 product photos
    → Shipping cost (fixed amount — passed through 100% to vendor)

Step 4: Submit for Admin Review
    → Admin reviews product photos, description, pricing
    → Approve: product goes live
    → Reject: reason sent to vendor with correction request
```

### 2.4 Vendor Operations Guide

**Managing Orders:**
1. You receive a push notification when an order is placed
2. Log in and confirm you have the item in stock
3. Ship within **3 business days** (SeaConnect SLA)
4. Enter the tracking number in the app: Orders → [Order ID] → Mark as Shipped
5. Customer is notified automatically

**Getting Paid:**
- Payout is sent 7 days after the order is marked as delivered (or after customer confirms receipt)
- If customer doesn't confirm within 7 days, payout is auto-released
- Payout = product price - 10% commission - ~2.5% Fawry fee + shipping (100% passed through)

**Handling Returns:**
- Customer has 14 days to report a damaged or wrong item
- Submit a dispute via the app → admin mediates
- If dispute is valid, refund issued and vendor payout withheld
- Vendor is responsible for return shipping on their error

## Part 3: Admin Onboarding & Approval Process

### 3.1 Admin Portal Access

The admin portal is a Next.js web app at `admin.seaconnect.app`.

Admin roles:
- **Super Admin** (co-founders): full access
- **Operations Admin**: approvals, user management, bookings
- **Support Admin**: view-only, handle disputes and refunds

### 3.2 Yacht Listing Approval Process

When a new boat listing is submitted:
1. Admin receives email: "New boat listing pending review: [Boat Name]"
2. Open admin portal → Approvals → Boats
3. Review:
   - Photos (minimum quality, no placeholder images)
   - Documents (check for validity, expiry dates)
   - Pricing (sanity check — not obviously wrong)
   - Description (no spam, not offensive)
4. Approve → Listing goes live → Owner notified
5. Reject → Select reason(s) from dropdown → Owner notified with reason

**SLA:** All listings reviewed within **24 hours** of submission.

**Document Verification Checklist:**
- [ ] Vessel registration number matches the boat described
- [ ] Insurance coverage amount is reasonable (minimum 500,000 EGP recommended)
- [ ] Documents are not expired
- [ ] Documents are legible (not blurry photos)
- [ ] Captain's license matches the captain named in the listing

### 3.3 Product Approval Process

When a new product is submitted:
1. Admin portal → Approvals → Products
2. Review:
   - Photos (real product photos, no watermarks, no stock images)
   - Description (accurate, in Arabic or English)
   - Price (reasonable for the category)
   - Category (correctly classified)
3. Approve → Product live → Vendor notified
4. Reject → Reason sent

**SLA:** Products reviewed within **24 hours**.

**Common Rejection Reasons:**
- Blurry or misleading photos
- Counterfeit brand goods
- Prohibited items (live animals, weapons, chemicals)
- Price clearly unrealistic (too high or obvious placeholder)
- Missing Arabic description (bilingual required)

### 3.4 Vendor Profile Approval

When a new vendor registers:
1. Review store profile: store name, description, logo
2. Verify no duplicates (same store registered twice)
3. Approve vendor profile (separate from product approval)
4. Vendor can then submit products once store is approved

## Part 4: Customer Onboarding

Customers register and book without admin approval. Flow:
```
Download app → Register (email or phone or Google)
→ OTP verification
→ Select "I'm a Customer"
→ Browse listings immediately (no further steps required)
```
